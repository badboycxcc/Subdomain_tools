# Subdomain Tools 全流程逻辑说明

## 1. 入口与任务类型
- 子域名任务：只做子域名收集与去重。
- 反查任务：输入 IP，收集关联域名并去重。
- 全流程任务：子域名收集 + 被动 IP + Web 探测 + 汇总输出。

## 2. 全流程（Pipeline）详细步骤
### 步骤 1/4：收集子域名
- 输入根域名（例如 `ocm.gov.kh`）。
- 并发调用子域名源：
  - `subfinder_sdk`
  - `ipthc_subdomain`
  - `hackertarget_hostsearch_subdomain`
  - `urlscan`
  - `myssl`
  - `rapiddns`（配置 API Key 后启用）
- 主要接口：
  - `https://api.hackertarget.com/hostsearch/?q={domain}`
  - `https://urlscan.io/api/v1/search/?q=domain:{domain}`
  - `https://myssl.com/api/v1/discover_sub_domain?domain={domain}`
  - `https://rapiddns.io/api/search/{domain}?page=1&pagesize=100&search_type=subdomain`
- 认证头：
  - `X-API-KEY: rdns_xxx`
- 每个源结束后输出日志：
  - `xxx 子域名获取 N 条，新增 M 条，累计去重 K 条`
- 阶段结束日志：
  - `xxx.com 子域名收集完成，共 K 个`

### 步骤 2/4：归类 API 查询 IP + dnsx 解析 A 记录
- 将各子域名接口返回的 IP 统一归类为 `API查询IP`（按 `host -> ip[]` 归档）。
- 补充调用 `hackertarget_hostsearch` 等 Host-IP 接口，继续并入 `API查询IP`。
- 程序内置 `dnsx` 风格 A 记录解析能力，对全部子域名查询当前解析 IP（无需本地命令，可在 Settings 配置 DNS 解析器）。
- 阶段日志：
  - `API 查询 IP 归类完成：域名 N 个，IP M 个`
  - `dnsx A 记录查询完成：命中 X/Y`

### 步骤 3/4：Web 探测与指纹识别
- 对每个子域名执行探测：
  - `dnsx` 查询到公网 A 记录：直接探测（`web_probe`）。
  - `dnsx` 无记录或仅内网/保留地址：走 `hosts_collision`。
  - `hosts_collision` 候选 IP 使用“收集到的全部 IP 列表”（`API查询IP + dnsx 结果池`）。
- 探测命中后输出两类内容：
  - 结果区：`[Web探测] URL=... | 状态=... | 标题=... | 技术=...`
  - 日志区：`命中结果：URL=... | 状态=... | 标题=... | 技术=...`
- 所有命中按 `host + url` 去重。

### 步骤 4/4：汇总结果
- 统计并输出最终计数：
  - `全流程完成：探测原始 X 条，去重 Y 条`

## 3. 去重与输出规则
- 计数维度：
  - 原始：所有抓取/探测尝试累计数。
  - 去重：按去重键合并后的唯一结果数。
- 子域名任务展示维度：
  - `Subdomains` 面板：仅显示去重后的域名结果（只显示首次出现）。
  - `Logs` 面板：显示全部过程明细（包含重复项、来源、阶段日志；若接口返回 IP，则记录 `IP=...`）。
  - 重复项在日志中标记为：`[重复]`。
- Web 探测展示维度：
  - `Subdomains` 面板：显示去重后的命中结果。
  - `Logs` 面板：显示命中明细与过程日志。

## 4. 界面输出约定
- 日志区：中文、简洁、按时间顺序展示每一步。
- 子域名输出区：仅展示去重后域名列表，便于直接复制使用。
- 过程细节（包括重复项）统一在日志区展示。
- 统计栏：实时显示 `原始: X / 去重: Y`。
